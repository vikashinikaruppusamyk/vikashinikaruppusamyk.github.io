To-Do / Task Management App

Overview

A production-grade full-stack task management application demonstrating modern backend architecture patterns. The system handles user authentication, task CRUD operations, and role-based access control — all within a clean Spring Boot backend.

Problem Statement

Build a secure, scalable task management API that supports multiple users with isolated data, stateless authentication, and protected endpoints — while keeping the codebase maintainable and extensible.

Architecture
  • Controller → Service → Repository layered architecture for clean separation of concerns
  • JWT-based stateless auth — no server-side session state, tokens carry user identity and roles
  • Spring Security for endpoint protection with role-based access control
  • Normalised PostgreSQL schema with JPA/Hibernate for ORM — entities mapped with proper relationships
  • Vanilla JS frontend consuming the REST API as a separate client layer
Key Engineering Decisions
  • Chose JWT over sessions for stateless scalability — each request is self-contained
  • Used Spring Security's filter chain to intercept and validate tokens before hitting controllers
  • Designed schema with foreign keys and constraints to enforce data integrity at the DB layer
  • Structured DTOs separately from entities to decouple API surface from DB schema
Tech Stack
Spring Boot PostgreSQL JWT Spring Security JPA / Hibernate REST APIs Maven Vanilla JS

Library Management System

Overview

A fully OOP-driven library management system that demonstrates mastery of classic design patterns and SOLID principles — built in pure Java without any web framework, to show strength at the language level.

Problem Statement

Model a real-world library with inventory control, patron management, borrowing workflows, and automated notifications — all in a way that's extensible, testable, and following best OOP practices.

Design Patterns Applied
  • Factory Pattern — for creating different types of library items (books, magazines) without coupling callers to concrete classes
  • Observer Pattern — automated notifications to patrons when reserved books become available
  • Repository Pattern — abstracted data access layer that separates persistence logic from business logic
Key Features
  • Inventory management — add, remove, update, and search books by multiple criteria
  • Patron lifecycle tracking — borrow, return, and overdue management
  • File-based persistence — data survives restarts without a database dependency
  • Structured logging with SLF4J for production-style observability
Tech Stack
Java 25 OOP Design Patterns SOLID SLF4J File I/O

LearnTrack — Learning Management System

Overview

A Learning Management System REST API built with Spring Boot that handles course lifecycle, student enrollment, and progress tracking — designed as a production-style backend service with a clean layered architecture.

Problem Statement

Build a multi-entity backend system where instructors can create courses, students can enroll and track progress, and the system enforces business rules (e.g., enrollment limits, course status transitions).

Architecture
  • Controller → Service → Repository layers with strict boundaries — no business logic leaking into controllers
  • JPA entities with proper relationship mappings (OneToMany, ManyToMany for enrollments)
  • Service layer enforces business rules — enrollment status transitions, capacity checks
  • REST endpoints follow resource-oriented design with correct HTTP verbs and status codes
Key Features
  • Course management — create, activate, deactivate with status lifecycle
  • Student management — registration, profile, deactivation
  • Enrollment system — enroll, cancel, complete with status tracking
  • Search and filter endpoints for both courses and students

MediTrack — Clinic Appointment System

Overview

A clinic appointment scheduling system built in pure Core Java — no Spring, no framework. The deliberate constraint demonstrates depth in Java fundamentals and the ability to reason about system design without relying on framework abstractions.

Problem Statement

Model a clinic's appointment workflow — patient registration, doctor availability, appointment booking, and cancellations — with persistent state across sessions using file I/O.

Technical Highlights
  • Java 8+ Streams for filtering, mapping, and aggregating appointment data functionally
  • File-based persistence — serialised state that survives restarts without a DB dependency
  • Comprehensive exception handling with meaningful error messages at every boundary
  • Domain-driven class design — separate entities, services, and I/O handlers
  • Java Collections framework used idiomatically for scheduling logic
Key Features
  • Patient registration and profile management
  • Doctor schedule and availability management
  • Appointment booking, rescheduling, and cancellation
  • Menu-driven CLI interface with input validation
